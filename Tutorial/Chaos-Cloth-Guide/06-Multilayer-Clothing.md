# 6. 멀티레이어 의상

## 6.1 멀티레이어란?

실제 캐릭터는 옷을 여러 겹 입습니다.

```
레이어 3 (외부):  망토/코트
레이어 2 (중간):  재킷/조끼
레이어 1 (내부):  셔츠/언더웨어

각 레이어가 독립적으로 시뮬레이션되면서도
서로 뚫지 않아야 합니다!
```

> **초보자 참고:** 멀티레이어는 중급~고급 주제입니다. 단일 레이어 (치마 하나, 망토 하나)가 잘 되는지 먼저 확인하고 진행하세요.

## 6.2 멀티레이어에서 생기는 문제

| 문제 | 설명 | 심각도 |
|------|------|:------:|
| **레이어 간 관통** | 셔츠가 재킷을 뚫음 | 높음 |
| **레이어 압착** | 의상이 찌그러져 한 겹처럼 보임 | 중간 |
| **늘어남 (Stretching)** | 레이어 간 압축으로 천이 늘어남 | 중간 |
| **성능 폭발** | 레이어 수 × 파티클 = 연산량 급증 | 높음 |

```
[문제 예시]                    [해결 후]

  재킷이 셔츠를 뚫음             재킷이 셔츠 위에 자연스럽게
  ┃══╬══┃  ← 관통!            ┃──┃──┃  ← 간격 유지
  ┃  ╬  ┃                     ┃  ┃  ┃
```

## 6.3 해결 전략 1: Collision Thickness 차등

가장 간단한 멀티레이어 대응법입니다.

### 원리

바깥 레이어일수록 **Collision Thickness를 크게** 설정하면,
안쪽 레이어와 자연스럽게 간격이 생깁니다.

| 레이어 | Collision Thickness | 캡슐 크기 |
|--------|:-------------------:|----------|
| 레이어 1 (셔츠) | 2.5~3mm (기본값) | 몸에 맞게 |
| 레이어 2 (재킷) | **4~5mm** | 셔츠보다 크게 |
| 레이어 3 (망토) | **5~8mm** | 가장 크게 |

```
      몸   셔츠  재킷  망토
      ┃    │    │     │
      ┃ 2.5│ 5mm│ 8mm │
      ┃    │    │     │
```

### 설정 방법

```
각 Cloth Asset마다:
  └─ Cloth Config → Collision Thickness Outer 값 변경
  └─ 바깥 레이어일수록 값을 키움
```

## 6.4 해결 전략 2: 격리된 콜리전 세트

각 레이어마다 **다른 Physics Asset 캡슐 세트**를 사용합니다.
(4장에서 소개한 개념의 확장)

### 핵심: 바깥 레이어일수록 캡슐을 크게

```
레이어 1: 셔츠
└── Collision Set: 몸통 캡슐만 (작은 크기)
    ├── spine_01~03: Capsule (몸에 딱 맞게)
    └── upperarm_l/r: Capsule (팔에 딱 맞게)

레이어 2: 재킷
└── Collision Set: 셔츠 + 여유분
    ├── spine_01~03: Capsule (셔츠보다 크게!)
    ├── upperarm_l/r: Capsule (셔츠보다 크게!)
    └── neck_01: Capsule

레이어 3: 망토
└── Collision Set: 재킷 + 여유분
    ├── spine_01~03: Capsule (가장 크게!)
    ├── head: Sphere
    └── upperarm_l/r: Capsule (가장 크게!)
```

> 레이어를 쌓을 때마다 캡슐 크기를 **누적으로** 키운다고 생각하세요.

## 6.5 해결 전략 3: 내부 레이어 간소화

성능을 위해 **보이지 않는 내부 레이어는 단순하게** 만듭니다.

| 최적화 | 효과 |
|--------|------|
| 내부 레이어 파티클 수 줄이기 | 연산량 대폭 감소 |
| 내부 레이어 Self Collision 끄기 | 큰 성능 절약 |
| 내부 레이어 Iteration 줄이기 | 비용 감소 |
| 원거리(LOD 1+)에서 내부 레이어 비활성화 | 거리별 최적화 |

```
[풀 시뮬레이션]                [최적화 후]

  망토: 1000 파티클              망토: 500 파티클 (외부)
  재킷: 800 파티클               재킷: 200 파티클 (간소화)
  셔츠: 600 파티클               셔츠: 100 파티클 (최소한)
  ─────────────                 ─────────────
  총: 2400 파티클                총: 800 파티클 (67% 절약!)
```

## 6.6 Outfit Asset (UE 5.6+)

UE 5.6에서 도입된 **Outfit Asset**은 멀티레이어를 에셋 단위로 관리합니다.

### 주요 기능

| 기능 | 설명 | 예시 |
|------|------|------|
| **에셋 묶기** | 여러 Cloth Asset을 하나의 Outfit으로 | "기사 갑옷" = 셔츠+체인메일+타바드+망토 |
| **리사이징** | 다양한 체형에 자동 크기 조절 | 같은 갑옷을 뚱뚱한/마른 캐릭터에 |
| **리피팅** | 체형 변화 시 핏 재조정 | 모프 타겟으로 체형 변할 때 |
| **레이어 순서** | 내부→외부 순서 명시 | Layer 0, 1, 2, 3 |

### 사용 예시

```
Outfit Asset: "KnightArmor"
├── Layer 0: Undershirt (셔츠) — 파티클 적음, 간소
├── Layer 1: Chainmail (체인메일) — 중간
├── Layer 2: Tabard (타바드) — 중간
└── Layer 3: Cape (망토) — 풀 시뮬, Self Collision

→ 다른 체형 캐릭터에 적용 시 자동 리사이징!
```

> **UE 5.6 미만 사용자:** Outfit Asset은 사용할 수 없지만, Collision Thickness 차등 + 격리된 콜리전 세트 전략으로 동일한 효과를 낼 수 있습니다.

### Cloth-to-Cloth Constraints (UE 5.6+)

5.6에서 추가된 기능으로, **천과 천 사이에 직접 컨스트레인트**를 걸 수 있습니다.

| 용도 | 설명 |
|------|------|
| 재킷 위 벨트 | 벨트가 재킷 위에 고정됨 |
| 어깨 갑옷 위 망토 | 망토가 갑옷 어깨에서 시작 |
| 다중 장신구 | 목걸이가 셔츠 위에 놓임 |

### Simulation Morph Target (UE 5.6+)

시뮬레이션 중에 **모프 타겟**을 적용할 수 있습니다.

| 용도 | 설명 |
|------|------|
| 근육 팽창 | 근육이 부풀면 옷도 따라 팽창 |
| 체형 변화 | 살찐/마른 블렌드 시 옷 자동 조절 |
| 표정 연동 | 얼굴 주변 천(후드 등) 반응 |

## 6.7 초보자를 위한 멀티레이어 시작 가이드

처음 멀티레이어를 시도한다면:

```
Step 1: 단일 레이어 완성 (3~5장 내용)
        └─ 망토 하나가 잘 시뮬레이션되는지 먼저 확인
               ↓
Step 2: 두 번째 레이어 추가
        └─ 기존 단일 레이어 위에 하나 더 추가
        └─ Collision Thickness를 다르게 설정
               ↓
Step 3: 관통 확인
        └─ 다양한 애니메이션으로 테스트
        └─ 관통 발생 시 Thickness 또는 캡슐 크기 조절
               ↓
Step 4: 성능 확인
        └─ stat Cloth 명령어로 비용 체크
        └─ 내부 레이어 간소화 (파티클 줄이기)
```
