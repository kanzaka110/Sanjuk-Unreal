# 3. 핵심 개념 이해하기

## 3.1 FocalRig의 구조

FocalRig은 UE5의 **Control Rig** 시스템 위에서 동작하는 노드 기반 플러그인입니다. Control Rig이 무엇인지 먼저 이해하면, FocalRig을 훨씬 쉽게 배울 수 있습니다.

### Control Rig이란?

```
Control Rig = UE5의 프로시저럴 리깅 시스템

쉽게 말하면:
- 애니메이션을 "코드(노드)"로 제어하는 시스템
- 블루프린트처럼 노드를 연결하여 뼈대(Bone)를 조작
- 런타임에서 실시간으로 계산되므로, 상황에 맞게 반응 가능
```

### FocalRig의 위치

```
UE5 애니메이션 시스템 구조:

Skeletal Mesh (스켈레탈 메시)
  └── Animation Blueprint (애니메이션 블루프린트)
        ├── State Machine (상태 머신) → 걷기, 뛰기, 점프 등
        ├── Aim Offset (기존 방식) → ❌ FocalRig이 대체
        └── Control Rig (컨트롤 리그)
              └── FocalRig 노드들  ← ✅ 여기서 동작!
                    ├── Aim Chain
                    ├── Eye Aim
                    ├── Recoil
                    └── Leg Adjustment
```

## 3.2 Aim Chain (에임 체인) — 핵심 중의 핵심

### 개념

Aim Chain은 FocalRig의 **가장 중요한 기능**입니다. 캐릭터의 척추(Spine) 본들을 순차적으로 회전시켜, **시점 본(POV Bone)**이 **타겟**을 정확히 가리키도록 합니다.

### 작동 원리

```
                      타겟 (적, 오브젝트 등)
                        ★
                       ╱
                      ╱
    [머리] ←────────╱── POV Bone (시점 본)
    [목]           ╱
    [상부 척추]   ╱     ← 이 본들이 순차적으로
    [중부 척추]  ╱        약간씩 회전하여
    [하부 척추] ╱         자연스러운 커브를 만듦
    [골반]    ╱
    ─┬───┬─
     │   │
    [좌] [우]  ← 다리 (Leg Adjustment로 보정)
```

### 핵심 용어

| 용어 | 영어 | 설명 |
|------|------|------|
| 시점 본 | POV Bone | 타겟을 향해 정렬되는 최종 본 (보통 Head 또는 Weapon Bone) |
| 타겟 | Target | 캐릭터가 바라보거나 조준할 목표 위치 (월드 좌표) |
| 에임 체인 | Aim Chain | POV Bone까지 이어지는 본들의 연쇄 (Spine 계열) |
| 가중치 분배 | Weight Distribution | 각 본이 얼마나 회전할지 결정하는 비율 |

### 가중치 분배 이해하기

```
예시: 오른쪽으로 60도를 돌려야 할 때

가중치 분배 (기본값 예시):

  골반 (Pelvis):     10% → 6도 회전
  하부 척추 (Spine1): 15% → 9도 회전
  중부 척추 (Spine2): 20% → 12도 회전
  상부 척추 (Spine3): 25% → 15도 회전
  목 (Neck):         15% → 9도 회전
  머리 (Head):       15% → 9도 회전
  ────────────────────────────────────
  합계:              100%   60도

→ 각 본이 조금씩 나눠서 회전하므로 자연스러운 커브가 형성됨!
→ 한 본만 크게 도는 것보다 훨씬 리얼리스틱
```

## 3.3 Eye Aim (아이 에임) — 눈 추적 시스템

### 개념

캐릭터의 **눈이 타겟을 추적**하는 시스템입니다. 단순히 "눈을 타겟 방향으로 돌리는" 것이 아니라, **실제 사람의 눈 움직임**을 시뮬레이션합니다.

### 핵심 요소

#### 1) 스프링 댐핑 (Spring Damping)

```
눈이 타겟을 따라갈 때의 움직임 방식:

단순 추적:     눈이 즉시 타겟으로 → 로봇처럼 딱딱함
스프링 댐핑:   눈이 부드럽게 따라감 → 자연스러움
               약간 오버슈팅 후 안정화 → 살아있는 느낌

[타겟 이동] ──→ [눈이 살짝 늦게 따라감] ──→ [약간 지나침] ──→ [안정]
```

#### 2) Saccade (급속 안구 운동)

```
사카드(Saccade)란?

실제 사람의 눈은 한 곳을 응시할 때도 미세하게 떨립니다.
이것을 "급속 안구 운동" 또는 "Saccade"라고 합니다.

타겟을 응시 중:
  ┌─────────────────────────┐
  │         ★ 타겟          │
  │        ↗↙↘↖             │
  │     (미세한 떨림)        │
  │  = 살아있는 느낌!       │
  └─────────────────────────┘

FocalRig은 이 Saccade를 자동으로 시뮬레이션합니다.
```

#### 3) 각도 제한 & 최소 거리

```
각도 제한:
- 사람의 눈은 좌우 약 ±50도 정도만 움직일 수 있음
- FocalRig에서 이 범위를 설정 가능
- 범위를 벗어나면 머리/몸이 같이 회전 (Aim Chain과 연동)

최소 거리:
- 타겟이 너무 가까우면 양쪽 눈이 안쪽으로 몰림 (사팔뜨기)
- 최소 거리를 설정하면, 가까운 타겟은 무시하거나 자연스럽게 처리
- 기본값으로도 잘 작동하지만, 커스터마이즈 가능
```

## 3.4 Recoil (반동) 시스템

### 개념

무기 발사 시의 **반동 패턴**을 만들고 적용하는 시스템입니다. 무기뿐만 아니라 카메라의 흔들림에도 사용할 수 있습니다.

### 2D 패턴 에디터

```
FocalRig 반동 패턴 에디터:

     Y (상)
     ↑
     │  ┌─→ 3번째 반동
     │  │ ┌─→ 2번째 반동
     │  │ │
     │  2 3
     │ ╱
     │1        ← 1번째 반동
     │╱
     ○────────→ X (우)
   (발사점)

- 각 점은 반동으로 인한 이동 위치
- 점을 드래그 앤 드롭으로 편집
- 연사 속도에 맞춰 미리보기 가능
- 데이터 에셋으로 저장 → 재사용
```

### 내장 프리셋

| 프리셋 | 특징 | 반동 패턴 |
|--------|------|----------|
| SMG (기관단총) | 빠른 연사, 작은 반동 | 위로 살짝, 좌우 미세 |
| Assault Rifle (돌격소총) | 중간 연사, 중간 반동 | 위로 상승 후 좌우 퍼짐 |
| LMG (경기관총) | 느린 연사, 큰 반동 | 위로 크게, 좌우 크게 |
| Shotgun (산탄총) | 단발, 매우 큰 반동 | 위로 한번 크게 튐 |

### 데이터 에셋

```
반동 패턴은 "데이터 에셋"으로 저장됩니다:

장점:
1. 한번 만들면 여러 무기에 재사용 가능
2. 런타임에서 동적으로 교체 가능
3. Control Rig과 Blueprint 양쪽에서 사용 가능
4. 팀원과 공유 가능 (에셋 파일이므로)
```

## 3.5 Leg Adjustment (다리 보정)

### 개념

Aim Chain으로 상반신이 회전할 때, **발이 바닥에서 미끄러지는 것을 방지**합니다.

### 왜 필요한가?

```
다리 보정 없이 상반신만 회전하면:

회전 전:              회전 후:
    O                    O→ (상반신이 오른쪽으로 회전)
   ─┼─                 ─╱╲─
   ╱ ╲                 ╱   ╲
  ╱   ╲              ╱     ╲  ← 발이 원래 위치에서 벗어남!
 ─     ─            ─       ─    미끄러지는 것처럼 보임

다리 보정 적용 시:

회전 전:              회전 후:
    O                    O→
   ─┼─                 ─╱╲─
   ╱ ╲                 ╱ ╲   ← 발이 원래 위치에 고정!
  ╱   ╲              ╱   ╲     IK로 다리 각도를 자동 보정
 ─     ─            ─     ─
```

### 핵심 원리

```
Leg Adjustment는 내부적으로 이런 과정을 거칩니다:

1. 골반이 회전하기 전 발의 월드 위치를 기록
2. 골반 회전 적용
3. 회전으로 인해 발이 이동한 위치를 계산
4. IK(Inverse Kinematics)로 발을 원래 위치로 되돌림
5. 무릎 방향도 자동으로 조정

결과: 상반신이 아무리 돌아도 발은 바닥에 딱 붙어있음!
```

## 3.6 Quick Setup (빠른 설정)

### 개념

FocalRig의 모든 노드는 **Quick Setup** 기능을 가지고 있습니다. 노드를 추가하면 스켈레톤을 자동으로 분석하여 설정을 채워줍니다.

### 자동 설정 과정

```
1. Control Rig에 FocalRig 노드를 추가
         ↓
2. 노드가 연결된 스켈레톤을 자동 스캔
         ↓
3. 본 이름, 축(Axis), 각도 제한 등을 자동 감지
         ↓
4. Quick Setup 드롭다운에 추천 설정 목록 표시
         ↓
5. 첫 번째 설정이 자동 적용됨
         ↓
6. 필요에 따라 다른 설정 선택 또는 수동 조정
```

### 지원 스켈레톤

```
Quick Setup이 자동으로 감지하는 스켈레톤 유형:

✅ UE5 Mannequin (SK_Mannequin)
✅ MetaHuman
✅ Mixamo 캐릭터
✅ 대부분의 휴머노이드 스켈레톤
✅ 커스텀 스켈레톤 (본 이름이 표준적이면)

→ 광범위한 스켈레톤에서 자동으로 작동!
→ 자동 감지가 안 되면 수동으로 본을 지정하면 됨
```
