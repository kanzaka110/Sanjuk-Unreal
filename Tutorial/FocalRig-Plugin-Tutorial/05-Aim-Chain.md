# 5. Aim Chain 완전 정복

## 5.1 Aim Chain 심층 이해

3편에서 Aim Chain의 기본 개념을 배웠습니다. 이번 편에서는 Aim Chain의 **모든 설정 항목**을 하나하나 파헤쳐, 원하는 대로 커스터마이즈하는 방법을 배웁니다.

## 5.2 Aim Chain 노드의 모든 입력 핀

### 필수 입력

| 핀 이름 | 타입 | 설명 |
|---------|------|------|
| Execute | Exec | 실행 핀 (Forwards Solve에서 연결) |
| Target | Vector | 조준 대상의 월드 좌표 |

### 주요 설정 (Details 패널)

| 설정 | 타입 | 설명 | 기본값 |
|------|------|------|--------|
| Chain Bones | Bone Array | 체인에 포함할 본 목록 | Quick Setup이 채움 |
| POV Bone | Bone | 타겟을 향해 정렬할 최종 본 | head |
| Aim Axis | Axis | 조준 방향 축 (X, Y, Z, -X, -Y, -Z) | 자동 |
| Up Axis | Axis | 상방향 축 | 자동 |
| Alpha | Float | 전체 효과 강도 (0.0 ~ 1.0) | 1.0 |

## 5.3 체인 본(Chain Bones) 설정

### 어떤 본을 포함해야 하나?

```
일반적인 휴머노이드 Aim Chain:

최소 구성 (3개 본):
├── spine_03 (상부 척추)
├── neck_01 (목)
└── head (머리)

권장 구성 (5개 본):
├── spine_01 (하부 척추)
├── spine_02 (중부 척추)
├── spine_03 (상부 척추)
├── neck_01 (목)
└── head (머리)

풀 구성 (6개 본 + 골반):
├── pelvis (골반) — 선택사항
├── spine_01
├── spine_02
├── spine_03
├── neck_01
└── head
```

### 본 수에 따른 차이

```
본 3개: 상반신 위주로 빠르게 회전 → 민첩한 느낌
본 5개: 자연스러운 전신 회전 → 균형 잡힌 느낌 (권장)
본 6개+: 골반부터 회전 → 몸 전체가 따라감, 가장 리얼리스틱
```

> 💡 Quick Setup이 스켈레톤에 맞게 자동으로 최적의 본 목록을 선택해줍니다. 대부분의 경우 자동 설정 그대로 사용해도 좋습니다.

## 5.4 가중치(Weight) 상세 설정

### 본별 가중치 조정

각 체인 본마다 **개별 가중치**를 설정할 수 있습니다. 가중치는 해당 본이 전체 회전 중 얼마나 기여하는지를 결정합니다.

```
가중치 예시 비교:

[균일 분배]           [상반신 강조]        [머리 강조]
pelvis:  0.15         pelvis:  0.05        pelvis:  0.0
spine_01: 0.15        spine_01: 0.10       spine_01: 0.05
spine_02: 0.20        spine_02: 0.15       spine_02: 0.05
spine_03: 0.20        spine_03: 0.25       spine_03: 0.10
neck_01:  0.15        neck_01:  0.20       neck_01:  0.30
head:     0.15        head:     0.25       head:     0.50

결과:
- 균일: 전신이 고르게 움직임
- 상반신 강조: 어깨/가슴이 많이 돌아감
- 머리 강조: 머리가 먼저 빠르게 타겟을 봄
```

### 게임 장르별 권장 가중치

| 장르 | 골반 | 하부 척추 | 중부 척추 | 상부 척추 | 목 | 머리 |
|------|------|----------|----------|----------|-----|------|
| TPS 슈터 | 0.10 | 0.15 | 0.20 | 0.25 | 0.15 | 0.15 |
| FPS | 0.00 | 0.05 | 0.10 | 0.30 | 0.25 | 0.30 |
| 액션 RPG | 0.15 | 0.15 | 0.20 | 0.20 | 0.15 | 0.15 |
| 시네마틱 | 0.05 | 0.10 | 0.15 | 0.20 | 0.25 | 0.25 |

## 5.5 각도 제한 (Angle Limits)

### 왜 각도 제한이 필요한가?

```
각도 제한 없이:
- 캐릭터가 뒤를 돌아보려고 할 때
- 척추가 180도 이상 꺾일 수 있음
- → 엑소시스트처럼 부자연스러운 포즈!

각도 제한 적용:
- 각 본의 최대 회전 각도를 제한
- 사람의 실제 관절 가동 범위에 맞춤
- → 자연스러운 범위 내에서만 회전
```

### 본별 권장 각도 제한

| 본 | Yaw (좌우) | Pitch (상하) | Roll (기울임) |
|-----|-----------|-------------|-------------|
| pelvis | ±15° | ±5° | ±5° |
| spine_01 | ±20° | ±10° | ±5° |
| spine_02 | ±25° | ±15° | ±5° |
| spine_03 | ±30° | ±20° | ±10° |
| neck_01 | ±40° | ±30° | ±10° |
| head | ±50° | ±40° | ±15° |

> 💡 Quick Setup이 스켈레톤 분석을 통해 적절한 각도 제한을 자동 설정합니다. 자동 값으로 시작한 뒤 필요에 따라 미세 조정하세요.

## 5.6 Aim Axis와 Up Axis

### Aim Axis (조준 축)

```
Aim Axis = POV Bone이 타겟을 향해 정렬할 때 사용하는 축

스켈레톤마다 "앞"이 다를 수 있음:

UE5 Mannequin:  X가 앞  → Aim Axis = X
Mixamo:         -Y가 앞 → Aim Axis = -Y
커스텀:          다양    → 확인 필요

Quick Setup이 자동 감지하지만, 캐릭터가 엉뚱한 방향을 보면
Aim Axis를 변경해보세요.
```

### Up Axis (상방향 축)

```
Up Axis = 회전할 때 "위"가 어디인지 알려주는 축

일반적으로:
- UE5: Z-Up → Up Axis = Z
- 대부분 자동으로 올바르게 설정됨

잘못 설정하면:
- 캐릭터가 옆으로 누운 채 타겟을 봄
- 또는 거꾸로 뒤집힌 것처럼 보임
```

## 5.7 Alpha와 블렌딩

### Alpha 활용법

Alpha는 Aim Chain의 **전체 효과 강도**를 조절합니다.

```
Alpha = 0.0: Aim Chain 미적용 (원래 애니메이션 그대로)
Alpha = 0.5: 50%만 적용 (부드러운 효과)
Alpha = 1.0: 100% 적용 (완전한 에이밍)
```

### 상황별 Alpha 전환

```
게임 플레이 시나리오:

[일반 이동]     Alpha = 0.0  → 에이밍 없음
      ↓ (무기 조준 시작)
[에이밍 전환]   Alpha = 0.0 → 1.0 (0.3초에 걸쳐 보간)
      ↓
[조준 중]       Alpha = 1.0  → 완전 에이밍
      ↓ (조준 해제)
[에이밍 해제]   Alpha = 1.0 → 0.0 (0.2초에 걸쳐 보간)
      ↓
[일반 이동]     Alpha = 0.0
```

### Blueprint에서 Alpha 제어

```
// 블루프린트 의사코드

Event Tick:
  If (IsAiming)
    CurrentAlpha = FInterpTo(CurrentAlpha, 1.0, DeltaTime, 5.0)
  Else
    CurrentAlpha = FInterpTo(CurrentAlpha, 0.0, DeltaTime, 8.0)
  
  Set Control Rig Variable "Alpha" = CurrentAlpha
```

> 💡 FInterpTo를 사용하면 Alpha 값이 부드럽게 전환됩니다. InterpSpeed가 높을수록 빠르게 전환됩니다.

## 5.8 다중 타겟 처리

### 타겟 전환

```
적 A를 보다가 적 B로 타겟을 바꿀 때:

방법 1: 즉시 전환 (딱딱함)
─────────────────────
Target = 적B의 위치  ← 바로 변경

방법 2: 보간 전환 (자연스러움) ← 권장!
─────────────────────────────
Target = VInterpTo(현재타겟, 적B위치, DeltaTime, 전환속도)
```

### 타겟이 없을 때

```
타겟이 없는 상황 처리:

방법 1: Alpha를 0으로 → 에이밍 해제
방법 2: 캐릭터의 전방을 타겟으로 설정 → 정면 응시
방법 3: 마지막 타겟 위치 유지 → 서서히 Alpha 감소
```

## 5.9 Aim Chain 디버깅

### 흔한 문제와 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 캐릭터가 타겟의 반대를 봄 | Aim Axis가 반대 | Aim Axis를 반전 (X → -X) |
| 캐릭터가 옆으로 누움 | Up Axis가 잘못됨 | Up Axis 변경 |
| 회전이 너무 과함 | 각도 제한이 없음 | Angle Limits 설정 |
| 전혀 움직이지 않음 | 실행 핀이 안 연결됨 | Forwards Solve → Aim Chain 연결 확인 |
| 애니메이션이 깨짐 | 체인 본 순서가 잘못됨 | 부모 → 자식 순서 확인 |
| 떨림(Jitter)이 있음 | 타겟이 매 프레임 변동 | 타겟 위치에 보간(Interp) 적용 |

### Control Rig 디버그 모드

```
디버그 방법:

1. Control Rig 에디터에서 "Preview" 활성화
2. 뷰포트에서 본의 회전을 실시간 확인
3. 각 본에 걸린 가중치가 시각적으로 표시됨
4. 타겟 위치를 드래그하여 실시간 테스트
```

## 5.10 고급 팁

### 두 개의 Aim Chain 사용

```
상반신과 하반신을 별도로 제어하고 싶을 때:

[Aim Chain 1] — 상반신 (spine_02 → head)
              — 타겟: 적 위치
              — Alpha: 1.0

[Aim Chain 2] — 하반신 (pelvis → spine_01)
              — 타겟: 이동 방향
              — Alpha: 0.5

결과: 상반신은 적을 조준하면서, 하반신은 이동 방향을 향함
→ 전형적인 TPS 슈터의 움직임!
```

### Aim Chain + 기존 애니메이션 레이어링

```
Control Rig 연결 순서 (AnimGraph):

[Locomotion SM] → [Aim Offset (선택)] → [Control Rig (FocalRig)] → [Output]

또는 Aim Offset을 완전히 제거:

[Locomotion SM] → [Control Rig (FocalRig)] → [Output]
                   ↑
            이것만으로 충분!
```

## 체크리스트

다음 단계로 넘어가기 전에 확인하세요:

- [ ] 체인 본 구성을 이해하고 원하는 본을 선택할 수 있음
- [ ] 가중치 분배를 조정하여 원하는 느낌을 낼 수 있음
- [ ] 각도 제한을 설정하여 부자연스러운 포즈를 방지할 수 있음
- [ ] Alpha 값을 블루프린트에서 동적으로 제어할 수 있음
- [ ] 타겟 전환 시 보간을 적용할 수 있음
- [ ] 디버깅 방법을 알고 있음
