# 7. Recoil(반동) 시스템

## 7.1 Recoil 시스템 개요

FocalRig의 Recoil 시스템은 무기 발사 시의 반동을 **데이터 기반으로 제어**합니다. 2D 패턴 에디터에서 시각적으로 반동 패턴을 만들고, 이를 데이터 에셋으로 저장하여 다양한 무기에 적용합니다.

```
기존 반동 구현 vs FocalRig Recoil:

[기존 방식]
- C++이나 BP에서 랜덤 값으로 카메라 흔들기
- 매번 코드를 수정해야 패턴 변경 가능
- 디자이너가 직접 조정하기 어려움

[FocalRig 방식]
- 2D 그리드에서 드래그 앤 드롭으로 패턴 제작
- 데이터 에셋으로 저장 → 코드 수정 없이 교체
- 디자이너가 직접 시각적으로 편집 가능
- 연사 속도별 미리보기 지원
```

## 7.2 Recoil 노드 추가

### Control Rig에서 설정

```
1. Control Rig 에디터 (CR_FocalRig_Character) 열기

2. Rig Graph 우클릭 → "Recoil" 검색

3. "FocalRig Recoil" 노드 추가

4. 실행 순서 연결:
   [Forwards Solve] → [Aim Chain] → [Eye Aim] → [Recoil]
                                                    ↑
                                              마지막에 실행!

5. Quick Setup 적용:
   - 반동이 적용될 본 자동 감지 (보통 Head 또는 POV Bone)
```

### 왜 가장 마지막에 실행하나?

```
실행 순서의 이유:

1. Aim Chain → 몸이 타겟을 향해 정렬
2. Eye Aim → 눈이 타겟을 추적
3. Recoil → 정렬된 상태에서 반동을 "덮어쓰기"

반동은 에이밍 위에서 적용되어야 자연스럽습니다.
만약 반동이 먼저 적용되면, Aim Chain이 반동을 상쇄해버립니다.
```

## 7.3 반동 패턴 데이터 에셋

### 데이터 에셋 생성

```
1. Content Browser에서 원하는 폴더로 이동
   (예: Content/Weapons/RecoilPatterns/)

2. 우클릭 → Miscellaneous → Data Asset

3. 클래스 선택: FocalRigRecoilPattern (또는 플러그인이 제공하는 클래스)

4. 이름 지정:
   - DA_Recoil_AssaultRifle
   - DA_Recoil_SMG
   - DA_Recoil_Shotgun
   (DA = Data Asset)

5. 더블클릭하여 에디터 열기
```

### 2D 패턴 에디터 사용법

```
패턴 에디터 레이아웃:

     Y축 (상)
     ↑
  +5 │
     │              5●
  +3 │          4●
     │      3●
  +1 │   2●
     │ 1●
   0 ○─────────────→ X축 (우)
     │
  -1 │
     │
  -3 │
     │
  -5 │

각 ● 은 발사 시퀀스의 반동 위치:
- 1번째 발사: 약간 위로
- 2번째 발사: 더 위로 + 약간 우측
- 3번째 발사: 계속 상승
- ...
```

### 패턴 편집 방법

```
기본 조작:
- 좌클릭 + 드래그: 반동 포인트 이동
- 우클릭: 반동 포인트 추가
- Delete: 선택한 포인트 삭제
- 마우스 휠: 줌 인/아웃
- 중클릭 드래그: 그리드 팬

미리보기:
- 에디터 하단의 "Preview" 버튼 클릭
- Fire Rate(연사 속도) 슬라이더 조정
- ▶ 재생 → 연사 시 반동 패턴을 애니메이션으로 확인
```

## 7.4 내장 프리셋 활용

### 프리셋별 특징

#### SMG (기관단총) 프리셋

```
특징:
- 빠른 연사 (600~900 RPM)
- 작은 반동 진폭
- 위로 서서히 상승
- 좌우 미세한 흔들림

패턴 (대략):
     ↑
     │  5 6 7
     │ 3 4
     │1 2
     ○──→

적합한 무기: MP5, UMP, P90 등
```

#### Assault Rifle (돌격소총) 프리셋

```
특징:
- 중간 연사 (600~700 RPM)
- 중간 반동 진폭
- 처음 3발은 위로 상승
- 이후 좌우로 퍼지기 시작

패턴 (대략):
     ↑
     │      7
     │  5 6
     │ 3  4
     │ 2
     │1
     ○──→

적합한 무기: M4, AK-47, SCAR 등
```

#### LMG (경기관총) 프리셋

```
특징:
- 느린 연사 (500~600 RPM)
- 큰 반동 진폭
- 크게 위로 상승
- 좌우로 넓게 흔들림

패턴 (대략):
     ↑
     │        7
     │   5
     │      6
     │  3 4
     │ 2
     │1
     ○──→

적합한 무기: M249, PKM, MG3 등
```

#### Shotgun (산탄총) 프리셋

```
특징:
- 단발 또는 매우 느린 연사
- 한번에 매우 큰 반동
- 빠른 복귀 (리커버리)

패턴 (대략):
     ↑
     │
     │
     │
     │1
     │
     ○──→
   (한번 크게 튀었다가 복귀)

적합한 무기: 모스버그, 레밍턴, 베넬리 등
```

## 7.5 커스텀 반동 패턴 만들기

### Step-by-Step 가이드

```
예시: 커스텀 돌격소총 패턴 제작

1. 새 데이터 에셋 생성: DA_Recoil_CustomAR

2. 패턴 에디터에서:

   a. 1번 포인트 배치: (0, 1.5)
      → 첫 발은 위로 살짝
   
   b. 2번 포인트: (0.2, 3.0)
      → 두 번째 발은 더 위로, 약간 우측
   
   c. 3번 포인트: (-0.3, 4.0)
      → 세 번째 발은 위로, 약간 좌측
   
   d. 4번 포인트: (0.5, 4.5)
      → 네 번째 발은 우측으로 치우침
   
   e. 5번 포인트: (-0.2, 5.0)
      → 다섯 번째 발부터 좌우 패턴 시작
   
   f. 6~10번: 좌우로 왔다갔다 하며 서서히 상승

3. 미리보기에서 Fire Rate = 600 RPM으로 설정

4. ▶ 재생하여 패턴 확인

5. 만족스러우면 저장
```

### 패턴 디자인 팁

```
좋은 반동 패턴의 특징:

✅ 처음 3~5발은 예측 가능 → 숙련 플레이어 보상
✅ 이후 좌우 흔들림 증가 → 긴 연사 페널티
✅ 복귀(Recovery) 설정 → 발사 중지 시 돌아옴
✅ 총기 특성에 맞는 크기 → SMG < AR < LMG

❌ 피해야 할 패턴:
- 완전 랜덤 (학습 불가)
- 완전 일직선 (너무 쉬움)
- 너무 큰 진폭 (조작 불가)
```

## 7.6 Recoil 노드 설정 상세

### 주요 설정 항목

| 설정 | 타입 | 설명 |
|------|------|------|
| Recoil Pattern | Data Asset | 사용할 반동 패턴 에셋 |
| Fire Rate | Float | 연사 속도 (RPM) |
| Amplitude Scale | Float | 반동 크기 배율 (1.0 = 기본) |
| Recovery Speed | Float | 반동 복귀 속도 |
| Apply To Bone | Bone | 반동을 적용할 본 |

### Amplitude Scale (반동 크기 배율)

```
같은 패턴에서 크기만 변경:

Scale = 0.5  → 반동이 절반으로 줄어듦 (보정 장착 시)
Scale = 1.0  → 기본 반동
Scale = 1.5  → 반동이 1.5배 (조준경 사용 시)
Scale = 2.0  → 반동이 2배 (벽사 등)

활용: 어태치먼트(총기 부착물)에 따라 스케일 조정!
- 수직 손잡이 장착 → Scale × 0.7
- 머즐 브레이크 장착 → Scale × 0.8
- 없음 → Scale × 1.0
```

### Recovery Speed (복귀 속도)

```
발사를 멈추면 반동이 원래 위치로 돌아오는 속도:

Recovery = 1.0  → 느리게 복귀 (2~3초)
Recovery = 5.0  → 보통 속도로 복귀 (0.5~1초)
Recovery = 10.0 → 빠르게 복귀 (0.2초 이내)

권장:
- 슈터 게임: 5.0 ~ 8.0
- 리얼리즘 슈터: 2.0 ~ 4.0
- 아케이드 슈터: 8.0 ~ 15.0
```

## 7.7 Blueprint에서 Recoil 제어

### 발사 시 반동 트리거

```
블루프린트 로직 (의사코드):

Event: OnFire (무기 발사 이벤트)
│
├── 1. Control Rig에 "Fire" 이벤트 전달
│      → Recoil 노드가 다음 반동 포인트를 적용
│
├── 2. 현재 발사 카운터 증가
│      → 패턴의 몇 번째 포인트를 사용할지 결정
│
└── 3. 연사 중이면 반복, 아니면 Recovery 시작

Event: OnStopFire (발사 중지)
│
├── 1. 발사 카운터 리셋
│
└── 2. Recovery 시작 → 반동이 서서히 원위치로
```

### 무기 교체 시 패턴 변경

```
Event: OnWeaponSwap (무기 교체)
│
├── 1. 새 무기의 반동 패턴 데이터 에셋 가져오기
│      예: DA_Recoil_SMG → DA_Recoil_Shotgun
│
├── 2. Control Rig의 Recoil Pattern 변수 업데이트
│
├── 3. Fire Rate 업데이트
│
└── 4. Amplitude Scale 업데이트 (어태치먼트 반영)
```

## 7.8 카메라 반동

### Recoil을 카메라에 적용

```
FocalRig Recoil은 본(Bone)에 적용되지만,
카메라에도 동일한 패턴을 적용할 수 있습니다:

방법 1: FPS에서 Head Bone에 카메라가 붙어있을 때
────────────────────────────────────────────────
→ Recoil이 Head Bone을 움직이면 카메라도 같이 움직임
→ 별도 설정 불필요!

방법 2: TPS에서 카메라가 별도일 때
──────────────────────────────────
→ Recoil 패턴의 값을 읽어서 Camera Shake에 전달
→ 또는 별도의 Camera Recoil 로직 구현

방법 3: Blueprint에서 패턴 값 활용
──────────────────────────────────
→ Recoil 데이터 에셋에서 현재 오프셋 값을 읽어
→ Add Controller Pitch/Yaw Input으로 전달
```

## 7.9 Recoil 디버깅

### 흔한 문제와 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 반동이 전혀 없음 | Fire 이벤트가 안 전달됨 | BP에서 이벤트 연결 확인 |
| 반동이 너무 큼 | Amplitude Scale이 너무 높음 | Scale을 1.0으로 리셋 |
| 반동이 복귀하지 않음 | Recovery Speed가 0 | Recovery를 5.0 이상으로 |
| 패턴이 반복 안 됨 | 패턴 포인트가 부족 | 에디터에서 포인트 추가 |
| 방향이 반대 | Axis 설정 오류 | Apply 축 방향 확인 |

## 7.10 실전 레시피

### 레시피 1: 밸로런트 스타일 반동

```
패턴:
- 처음 5발: 직선 상승
- 6~15발: 좌우 지그재그
- 15발 이후: 랜덤성 증가

설정:
- Fire Rate: 680 RPM
- Amplitude Scale: 1.0
- Recovery Speed: 7.0
```

### 레시피 2: COD 스타일 반동

```
패턴:
- 전체적으로 작은 반동
- 약간의 상승 + 미세 좌우
- 빠른 복귀

설정:
- Fire Rate: 750 RPM
- Amplitude Scale: 0.6
- Recovery Speed: 12.0
```

### 레시피 3: 타르코프 스타일 반동

```
패턴:
- 크고 예측 가능한 상승
- 좌우 흔들림 적음
- 느린 복귀

설정:
- Fire Rate: 600 RPM
- Amplitude Scale: 1.8
- Recovery Speed: 3.0
```
