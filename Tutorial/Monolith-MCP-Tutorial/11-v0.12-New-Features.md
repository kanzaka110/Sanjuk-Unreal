## 2. 자동 재연결 프록시

v0.11.0부터 MCP 자동 재연결 프록시가 제공됩니다. UE 에디터를 재시작해도 Claude Code를 다시 시작할 필요가 없습니다.

### 설정 방법

`.mcp.json`을 프록시 방식으로 변경:

```json
{
  "mcpServers": {
    "monolith": {
      "command": "MonolithTest/Plugins/Monolith/Binaries/monolith_proxy.exe",
      "args": []
    }
  }
}
```

### 동작 원리

- 백그라운드 헬스 체크로 에디터 상태 자동 감지
- 에디터가 꺼져 있으면 도구 호출 시 세션을 죽이지 않고 에러 반환
- 에디터가 다시 켜지면 다음 호출부터 자동 연결
- `notifications/tools/list_changed`를 전송하여 Claude Code 도구 목록 자동 갱신

### v0.12.0 개선 (C++ 포팅)

| 항목 | 이전 (v0.11.0) | 현재 (v0.12.0) |
|------|---------------|----------------|
| 프록시 | `monolith_proxy.py` (Python) | `monolith_proxy.exe` (C++) |
| 의존성 | Python 3.8+ 필요 | 없음 |
| 크기 | — | 473KB |
| 시작 시간 | Python 인터프리터 로드 | 즉시 |

## 3. MonolithMesh — 메시 및 레벨 디자인 (242 액션)

Monolith에서 가장 큰 모듈. 레벨 디자인의 공간적 측면을 전면 커버합니다.

### 3.1 메시 검사 및 액터 CRUD

Claude에게 자연어로 요청:

```
"레벨에 있는 StaticMeshActor 목록을 보여줘"
"SM_Chair를 [500, 200, 0] 위치에 스폰해줘"
"선택한 액터를 Z축으로 90도 회전시켜"
"이 메시의 트라이앵글 수와 LOD 정보를 알려줘"
```

### 3.2 공간 쿼리 (PIE 없이 에디터에서 동작)

물리 기반 공간 쿼리를 에디터 모드에서 바로 사용할 수 있습니다:

```
"플레이어 스폰 위치에서 앞으로 1000 유닛 레이캐스트 해줘"
"이 방의 네비메시 커버리지를 확인해줘"
"반경 500 유닛 내에 겹치는 액터를 찾아줘"
```

### 3.3 레벨 블록아웃 워크플로우

프로토타이핑에 최적화된 블록아웃 → 최종 에셋 교체 파이프라인:

```
"3x4 크기의 방 블록아웃을 만들어줘"
"블록아웃 메시를 최종 에셋으로 일괄 교체해줘"
```

### 3.4 GeometryScript 메시 연산

```
"이 두 메시를 불리언 합집합으로 결합해줘"
"메시를 1000 트라이앵글로 단순화해줘"
"UV를 박스 프로젝션으로 다시 매핑해줘"
"LOD를 자동 생성해줘 (50%, 25% 비율)"
```

### 3.5 프로시저럴 지오메트리

파라메트릭 가구(15종), 건축 구조물, 미로(3가지 알고리즘), 파이프, 지형 패치 등을 자동 생성:

```
"테이블과 의자 4개를 배치해줘"
"3층 건물 구조를 생성해줘"
"20x20 크기의 미로를 만들어줘"
```

### 3.6 라이팅 및 오디오 분석

```
"이 레벨의 어두운 구석을 찾아줘"
"이 방의 RT60 잔향 시간을 계산해줘"
"스텔스 경로를 분석해줘"
```

### 3.7 성능 분석

```
"이 영역의 오버드로우를 분석해줘"
"그림자 비용이 높은 라이트를 찾아줘"
"텍스처 예산 리포트를 생성해줘"
```

### 3.8 Town Gen (실험적, 45 액션)

> 기본 비활성화. Editor Preferences에서 `bEnableProceduralTownGen` 활성화 필요.
> WIP 상태이므로 프로덕션 사용은 비권장.

```
"3x3 도시 블록을 생성해줘"
"이 건물에 파사드와 지붕을 추가해줘"
```

## 6. MonolithAI — AI 에셋 제어 (229 액션)

v0.12.0의 핵심 추가 기능. 15개 카테고리에 걸친 AI 시스템 전면 제어.

### 6.1 Behavior Tree (BT)

```
"순찰 AI용 Behavior Tree를 만들어줘"
"BT에 Sequence 컴포짓을 추가해줘"
"MoveTo 태스크를 추가하고 속도를 300으로 설정해줘"
"Wait 데코레이터를 2초로 추가해줘"
```

#### JSON 스펙으로 BT 일괄 생성 (핵심 기능)

`build_behavior_tree_from_spec`으로 JSON 하나로 완전한 BT를 만들 수 있습니다:

```
"다음 구조로 BT를 만들어줘:
 - Root > Selector
   - Sequence: 적 감지 → 추적 → 공격
   - Sequence: 순찰 포인트 이동 → 대기"
```

### 6.2 Blackboard

```
"TargetActor (Object), PatrolIndex (Int), IsAlerted (Bool) 키로 Blackboard를 만들어줘"
"Blackboard에 LastKnownLocation (Vector) 키를 추가해줘"
```

### 6.3 State Tree

```
"3개 상태(Idle, Patrol, Combat)의 StateTree를 만들어줘"
"Idle → Patrol 트랜지션 조건을 추가해줘"
"StateTree에 Evaluator를 추가해줘"
```

`build_state_tree_from_spec`으로 완전한 StateTree를 한번에 생성:

```
"Idle, Alert, Combat, Flee 4개 상태의 StateTree를 만들어줘.
 체력 30% 이하면 Flee로 전환"
```

### 6.4 EQS (Environment Query System)

```
"주변 커버 포인트를 찾는 EQS 쿼리를 만들어줘"
"SimpleGrid 제너레이터와 Distance 테스트를 추가해줘"
```

### 6.5 Smart Objects

## 8. C++ 독립 실행 도구

v0.12.0에서 Python 스크립트를 C++ 네이티브 실행 파일로 대체. Python 런타임 의존성이 완전히 제거되었습니다.

### monolith_proxy.exe

| 항목 | 설명 |
|------|------|
| 위치 | `Plugins/Monolith/Binaries/monolith_proxy.exe` |
| 크기 | 473KB |
| 역할 | MCP stdio-to-HTTP 프록시 |
| 기능 | JSON-RPC, 헬스 폴링, 도구 중복 제거, 액션 허용/차단 목록 |

### monolith_query.exe

| 항목 | 설명 |
|------|------|
| 위치 | `Plugins/Monolith/Binaries/monolith_query.exe` |
| 크기 | 1.8MB |
| 역할 | 오프라인 DB 쿼리 (에디터 없이 소스/프로젝트 검색) |
| 액션 | 소스 9개 + 프로젝트 5개 = 14개 |

오프라인 쿼리 사용 예:

```bash
# 에디터 없이 소스 코드 검색
monolith_query.exe search_source "ACharacter::BeginPlay"

# 에셋 검색
monolith_query.exe find_by_type "AnimSequence"
```

## 10. 옵셔널 모듈 시스템

v0.11.0부터 서드파티 플러그인 통합을 위한 옵셔널 모듈 시스템이 도입되었습니다.

### 동작 원리

- `Build.cs`에서 `Directory.Exists()`로 빌드 타임에 플러그인 존재 여부 감지
- `WITH_FOO=1` 또는 `=0` 전처리기 정의로 조건부 컴파일
- 플러그인이 없어도 DLL 로드 실패나 에러 없이 정상 컴파일

### 옵셔널 의존성 목록

| 플러그인 | Monolith 모듈 | 없을 때 |
|----------|--------------|---------|
| Blueprint Assist | MonolithBABridge | `auto_layout`이 내장 알고리즘 사용 |
| GBA (Blueprint Attributes) | MonolithGAS | BP AttributeSet 생성 불가, 나머지 130 액션 정상 |
| Logic Driver Pro | MonolithLogicDriver | 모듈 전체 비활성화 (빈 셸로 컴파일) |
| StateTree | MonolithAI | ST 관련 액션 비활성화 |
| SmartObjects | MonolithAI | SO 관련 액션 비활성화 |
| MassEntity | MonolithAI (선택) | Mass Entity 관련 액션 비활성화 |
